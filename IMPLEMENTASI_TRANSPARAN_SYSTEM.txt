1. LOGIN dari Client PC
   ───────────────────────────────────────────────────────────────────
   POST /api/client/session/login
   
   Request:
   {
     "pc_name": "PC-01",
     "member_id": 1,
     "duration_minutes": 60,
     "client_ip": "192.168.1.100",

   }
   
   Response (Success):
   {
     "success": true,
     "session": {
       "id": 1,
       "member": "John Doe",
       "pc": "PC-01",
       "status": "playing",
       "started_at": "2026-05-29T14:11:00+07:00",
       "duration_minutes": 60,
       "remaining_minutes": 60,
       "rate_per_minute": 500,
       "total_cost": 30000,
       "is_valid": true,
       "is_suspicious": false
     },
     "message": "Login successful"
   }
   
   Response (Error):
   {
     "success": false,
     "error": "Member sudah memiliki session aktif di PC lain"
   }

================================================================================
                     WORKFLOW SISTEM BARU
================================================================================

STEP 1: Member login dari Client PC
────────────────────────────────────────────────────────────────────
1. Member membuka aplikasi client GUI
2. Input username & password
3. Select PC (atau auto-detect)
4. Input durasi (berapa menit)
5. Client kirim POST /api/client/session/login
6. Server create PlaySession record dengan status 'playing'
7. Server update Computer status menjadi 'in_use'
8. Client terima session_id & tampilkan timer

STEP 2: Timer countdown di Client
────────────────────────────────────────────────────────────────────
1. Client kirim heartbeat setiap X detik
2. Server update last_heartbeat
3. Server update activity_log
4. Client display countdown (sisa waktu)
5. Member lihat real-time berapa lama lagi

STEP 3: Operator lihat di Dashboard
────────────────────────────────────────────────────────────────────
1. Buka halaman Computers
2. Lihat PC dengan status "🔵 Digunakan"
3. Lihat member name & waktu pakai
4. Bisa akhiri session manual jika perlu

STEP 4: Member logout
────────────────────────────────────────────────────────────────────
1. Member klik "Logout" di client
2. Client kirim POST /api/client/session/logout
3. Server call endSession()
   - Hitung actual duration
   - Hitung total cost
   - Log activity
   - Check suspicious activity
4. Server update Computer status menjadi 'available'
5. Session berakhir

STEP 5: Activity Log
────────────────────────────────────────────────────────────────────
Semua activity di-log ke JSON:
{
  "activity_log": [
    {
      "timestamp": "2026-05-29T14:11:00+07:00",
      "action": "login",
      "data": {"member": "John Doe", "duration": 60}
    },
    {
      "timestamp": "2026-05-29T14:12:00+07:00",
      "action": "heartbeat",
      "data": {"timestamp": "2026-05-29T14:12:00+07:00"}
    },
    {
      "timestamp": "2026-05-29T15:11:00+07:00",
      "action": "session_ended",
      "data": {"actual_duration_minutes": 60, "total_cost": 30000}
    }
  ]
}

================================================================================

System automatically detect fraud:

   └─ is_suspicious = true

2. ✅ Transparency total
   └─ Semua activity di log dengan timestamp
   └─ Member tidak bisa bohong

3. ✅ Real-time monitoring
   └─ Heartbeat detect kalau client offline
   └─ IP & MAC address tracked
   └─ Admin bisa review activity log
